1. 프로젝트 개요
크림(KREAM) 인기상품을 자동 수집하고, 무신사 등 쇼핑몰에서 더 싼 가격을 찾아 수익 기회를 디스코드로 알려주는 리셀 봇.

2. 완성된 기능
크롤링: 크림 인기상품 수집 (카테고리별: 신발/의류/가방/모자/액세서리/급상승), 크림 사이즈별 시세·거래량 수집, 무신사 상품 검색·가격 수집, CDP(Chrome) 로그인 + Pinia 폴백 방식

매칭 시스템: 3단계 검색 (1차: 복합 모델번호 슬래시 분리 검색, 2차: 상품명 한글 검색, 3차: 브랜드+상품명 검색), 모델번호 정확 일치만 매칭 인정 (부분 일치 거부), 매칭 실패/애매한 건은 #수정 채널에 사유와 함께 기록

수익 분석: 크림 수수료 반영 (6% + 검수비 2,500원 + 배송비 3,000원), 확정수익 (ROI 5% 이상) / 예상수익 (ROI 10% 이상) 구분, 무신사 품절(재입고 알림) 사이즈 자동 제외

디스코드 봇: 명령어: !자동스캔, !자동스캔시작, !자동스캔중지, !스캔, 30분 자동 반복 스케줄러 내장, 채널별 알림 분리 (#매수-알림, #수정, #일일리포트, #가격변동, #로그)

기타: Git 버전 관리 초기화 완료 (42파일, 9,553줄), HANDOFF.md 인수인계 문서 작성 완료, Chrome 경고는 터미널/로그에만 출력 (#매수-알림 채널 깨끗하게 유지)

4. .env 채널 설정 현황
CHANNEL_PRICE_CHANGE (가격변동), CHANNEL_DAILY_REPORT (일일리포트), CHANNEL_PROFIT_ALERT (매수알림), CHANNEL_BUY_ALERT (매수알림 중복이지만 문제없음), CHANNEL_MANUAL_SEARCH (수동검색 → 일반채널 ID), CHANNEL_LOG (로그 → 일반채널 ID), CHANNEL_MATCH_REVIEW (매칭검토 → #수정 채널 1487125534726160597) — 모두 설정 완료.

5. 알려진 문제점
카테고리 편향: 브랜드 키워드가 Nike/Adidas 등 신발 위주라 의류/가방/모자 수집이 부족함.

수집 범위 제한: 현재 카테고리별 인기상품 20~100개만 수집 → 크림 전체 상품 대비 극히 일부만 커버.

매칭률 낮음: 10개 중 2개만 매칭 (20%). 크림 인기상품이 한정판 위주라 무신사에 없는 경우가 많음.

쇼핑몰 1개만 지원: 현재 무신사만 검색. 29CM, ABC마트, 나이키공홈 등 미지원.

git 커밋 미완: 마지막 수정사항(카테고리 다변화 + 테스트)의 커밋이 한도 초과로 완료 안 됨.

6. 내일 할 일 (우선순위)
1순위 — git 커밋 마무리:

git add -A && git commit -m "카테고리 다변화 + 3단계 매칭 + 매칭검토 로그"
2순위 — 크림 전체 상품 DB 구축 방식으로 전환: 카테고리별 인기 100개 방식 → 크림 API로 전체 상품을 DB화. 품번/상품명/즉시구매가/즉시판매가/거래량 저장. !DB구축 명령어 추가. 매일 새벽 자동 업데이트.

3순위 — 양방향 스캔 구현: 정방향 (크림 DB → 무신사에서 싼 거 찾기) + 역방향 (무신사 세일 → 크림에서 비싸게 팔 수 있는 거 찾기) 동시 운영.

4순위 — 쇼핑몰 추가: 무신사 외 29CM, ABC마트, 나이키공홈 등 크롤러 추가 → 매칭률 상승.

5순위 — GitHub 원격 저장소 연동: 코드 백업 및 버전 관리.
