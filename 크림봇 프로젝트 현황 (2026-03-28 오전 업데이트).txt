크림봇 프로젝트 로드맵 & 전략 문서
1. 프로젝트 비전
크림(KREAM) 전체 상품을 DB화하고, 다수의 소싱처에서 자동으로 최저가를 찾아 수익 기회를 실시간 알림으로 제공하는 리셀 자동화 봇. 최종 목표는 캐찹매니아(월 20만원) 수준의 기능을 무료로, 더 나은 사용성으로 제공하는 것.

2. 현재 완성된 것 (2026.03.28 기준)
크림 API 연동 (인기상품 수집, 사이즈별 시세, 거래량), 무신사 크롤러 (상품 검색, 가격 수집, 품절 필터링), 3단계 매칭 시스템 (모델번호 분리 → 상품명 → 브랜드+상품명, 정확 일치만 인정), 디스코드 봇 (자동스캔, 수익 알림, 매칭 검토 로그, 30분 자동 반복), 크림 전체 상품 DB 구축 중 (카테고리별 페이지 단위 수집), 테스트 163개 통과, Git 버전 관리, HANDOFF.md 인수인계 문서.

3. 경쟁 분석 요약
캐찹매니아가 업계 1위. 월 20만원, 소싱처 134개(공홈 55 + 직매입 30 + 입점형 34 + 해외 15), 판매처 4개(크림/포이즌/솔드아웃/StockX), AI 품번 매칭, 색상 자동 매칭, 엑셀 출력, 대량 입찰 연동, 실시간 재고 모니터링. 우리 봇의 현재 GAP은 소싱처 1개(무신사만), 수집 범위 제한, 색상 매칭 없음, 엑셀 없음, 재고 감지 없음.

4. 우리만의 차별화 포인트
디스코드 실시간 푸시 알림 (캐찹매니아에 없음, 수익 기회 즉시 폰으로 수신), 웹 대시보드 (브라우저 접속, 모바일 지원 — 캐찹매니아는 Windows exe만), 가격 급변 감지 (캐찹매니아는 현재 가격만 비교, 우리는 추세 분석), 무료 (월 20만원 절약), 완전 커스터마이징 가능.

5. 개발 로드맵
Phase 1 — 엔진 완성 (현재~1주)

크림 전체 DB 구축 완료. JSON → SQLite 전환 (수만 개 상품 처리 속도). 소싱처 추가: 나이키공홈, 아디다스공홈, 29CM (총 4개). 병렬 검색 적용 (asyncio.gather로 4곳 동시 검색). 가격 히스토리 저장 시작. 목표: 실제 수익 1건 달성.

Phase 2 — 웹 대시보드 (2주차)

기술 스택: FastAPI(백엔드) + React/Next.js(프론트). 피그마에서 UI 디자인 → Claude Code로 코딩. 화면 구성: 메인 대시보드(DB 현황, 오늘 스캔 결과, 수익 기회 수), 상품 검색/필터(카테고리·브랜드·마진율), 상품 상세(소싱처별 가격 비교, 가격 추세 그래프, 사이즈별 마진), 스캔 관리(자동 ON/OFF, 주기 설정, 소싱처 선택), 수익 리포트(일별·주별·월별 통계). 엑셀 내보내기 기능 포함.

Phase 3 — 고급 기능 (3~4주차)

가격 급변 감지 (24시간 내 10% 이상 시세 상승 상품 자동 알림). 소싱처 확장: ABC마트, 카시나, 오케이몰 (총 7개). AI 품번 매칭 (표기 차이 자동 해결: DV0833-101 vs DV0833_101). 색상 자동 매칭. 사이즈 표기 변환 (US 9 = 270mm 등).

Phase 4 — 프로급 (1~2개월)

실시간 재고 모니터링 (소싱처 재고 변동 감지). 모바일 최적화 웹 대시보드. 입점형 마켓 추가 (쿠팡, SSG, 롯데온 등). 판매처 추가 (포이즌, 솔드아웃, StockX). 대량 입찰 연동 검토. 임대 서버 이전.

6. 소싱 전략 (4가지 병행)
전략 A — 정방향 (크림 → 소싱처): 크림 DB에서 거래량 많은 상품 → 소싱처에서 싼 거 찾기. 장점: 팔린다는 보장. 단점: 인기상품은 마진 적음.

전략 B — 역방향 (소싱처 → 크림): 소싱처 세일/할인 상품 → 크림 DB에서 비싸게 팔 수 있는지 확인. 장점: 할인 폭만큼 마진. 단점: 안 팔릴 위험.

전략 C — 가격 급변 감지: 크림 시세 급등 상품 감지 → 소싱처에서 아직 안 오른 거 빠르게 매입. 시간 싸움, 자동화 핵심.

전략 D — 전체 DB 대조: 크림 전체 상품 × 모든 소싱처 가격 비교 → 마진 필터링. 수집량 = 경쟁력.
