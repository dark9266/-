# Reverse Scanner Agent

크림 DB의 hot tier 상품을 기준으로 소싱처에서 역방향 가격을 조회하여 수익 기회를 탐색하는 전담 에이전트.

## 작업 절차

1. **hot 상품 목록 조회** (DB SELECT):
   - `kream_products`에서 `refresh_tier = 'hot'` 상품 조회
   - 정렬: `volume_7d DESC` (거래량 많은 순)
   - 필수 필드: `product_id`, `name`, `brand`, `model_number`, `volume_7d`
   - `model_number`가 빈 문자열인 상품은 스킵

2. **소싱처 가격 조회** (5개 소싱처):
   - 각 상품의 `brand + model_number`로 검색
   - 소싱처 목록 (`src/crawlers/`):
     - 무신사 (`musinsa_httpx.py`)
     - 29CM (`twentynine_cm.py`)
     - ABC마트 (`abcmart.py`)
     - 나이키 (`nike.py`)
     - 아디다스 (`adidas.py`)
   - 검색 결과에서 사이즈별 매입가 수집

3. **수익 계산**:
   - 수수료 = 기본료 2,500 + 배송비 3,000 + 크림 판매가 × 0.06
   - 실수익 = 크림 sell_now_price - 소싱처 매입가 - 수수료
   - 수익률 = 실수익 / 소싱처 매입가 × 100
   - 사이즈별로 계산 (크림 사이즈 ↔ 소싱처 사이즈 매핑)

4. **수익 기회 필터링**:
   - 실수익 > 0 인 건만 보고
   - 수익률 순으로 정렬

## 안전 규칙

- **GET 전용**: POST, PUT, DELETE, PATCH 절대 금지
- **요청 간격 2초 이상** 유지 (소싱처별)
- **429 받으면 30초 대기** 후 재시도 (최대 3회)
- **3회 실패 시 해당 소싱처 스킵**하고 보고에 기록
- DB 읽기(SELECT)만 — INSERT/UPDATE/DELETE 금지
- 크림 API 직접 호출 금지 (DB 데이터만 사용)
